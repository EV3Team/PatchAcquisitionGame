슬라이드 5: 1단계 - 알고리즘 설계
1) 행 우선 탐색 (Complete Search)

격자판을 좌→우, 우→좌 지그재그 방식으로 탐색 <br>
짝수 행: 왼쪽 → 오른쪽 <br>
홀수 행: 오른쪽 → 왼쪽 <br>
모든 격자점을 빠짐없이 방문

2) 동적 계획법 (DP)
dt[i][j] = 현재 위치까지의 최대 점수 <br>
dt[i][j] = max(dt[i-1][j], dt[i][j-1]) + S[i][j]

위 또는 왼쪽에서만 이동 가능 <br>
최적 부분 구조를 활용한 효율적인 경로 계산

3) 경로 역추적

DP 테이블을 역순으로 추적 <br>
더 높은 점수를 가진 경로 선택 <br>
출발점까지 최적 경로로 복귀

정확한 점수 출력 (5점)<br>
획득 점수: [실제 점수]점 (순위: [등수])

격자판 출력 예시 (실제 이동 경로를 선으로 간단하게 표기해주면 좋을듯!)<br>
```

+ O + +
+ X O +
+ + + X
+ O + +
  
```

+: 일반 격자점<br>
O: 빨간색 패치 (+1점)<br>
X: 파란색 패치 (-1점)

슬라이드 10: 2단계 과제 - 문제 정의<br>
과제 요구사항

격자판 크기: 4×5<br>
탐색 방식: 행 우선 탐색 + 자유 경로 이동<br>
복잡한 점수 체계:

빨간색 패치: 첫 방문 +5점, 재방문 -2점<br>
파란색 패치: 방문 시마다 -5점<br>
각 엣지 이동: -1점

난이도 증가 요소

자유 경로: 상하좌우 모든 방향 이동 가능<br>
방문 횟수: 같은 칸을 여러 번 방문 가능<br>
복잡한 점수 계산: 이동 비용 + 패치 점수 + 재방문 페널티

슬라이드 11: 2단계 - 알고리즘 설계<br>
깊이 우선 탐색 (DFS) + 백트래킹

목표: 이동점(3,4) → 출발점(0,0) 최대 점수 경로 탐색<br>
탐색 방법:

4방향(상하좌우) 모두 탐색<br>
백트래킹으로 모든 가능한 경로 시뮬레이션<br>
최대 점수 경로 저장

알고리즘 핵심 아이디어<br>
DFS(현재위치, 현재점수, 이동횟수):<br>
    1. 출발점 도착 시 → 최대 점수 갱신<br>
    2. 4방향 이동 가능성 확인<br>
    3. 각 방향으로 재귀 탐색<br>
    4. 백트래킹으로 다른 경로 탐색<br>
가지치기 조건

슬라이드 16: 기술적 도전과제 및 해결방법<br>
1) 라인 트레이싱 정확도

문제: 교차점에서 방향 전환 시 라인 이탈<br>
해결: 3개 센서 활용 + 타이밍 조정 (sleep 함수)

2) 패치 감지 오류

문제: 빠른 이동 시 패치 미감지<br>
해결: 교차점마다 추가 감지 루프 구현

3) DFS 성능 최적화

문제: 모든 경로 탐색 시 시간 초과<br>
해결:

최대 깊이 제한 (10칸)<br>
방문 횟수 제한 (2회)<br>
중복 방문 가지치기

4) 방향 전환 정확도

문제: 180도 회전 시 각도 오차<br>
해결: sleep 시간 정밀 조정 (1400ms)
